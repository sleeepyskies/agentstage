# [001] Use React

## Status - ACTIVE

## Context
For the Frontend of MadChatter, we should use a battle tested framework with reusable components, state management and fast rendering.
Some points we required:
- Strong ecosystem for third party libraries and extensions.
- Support for TypeScript.
- Easy to use and maintain.
- We should all either know the framework, or have an easy time getting started with it.

## Decided Approach

_Detail the architecturally significant decision that has been / will be made and describe how it
addresses the problems outlined in the Context section._
To solve this, we have decided to use the Frontend framework [React](https://react.dev/). Reasons include:
- Strong ecosystem and large community for external libraries/extensions.
- Battle tested framework.
- TypeScript support.
- We all have either experience with React, or with similar frameworks.
