# [003] Use shadcn

## Status - ACTIVE

## Context

The frontend requires a consistent UI system to ensure fast development and a cohesive user
experience. Building a component library from scratch would increase maintenance effort and lead to
design inconsistencies. A ready-made, extensible component system is needed to accelerate
development and enforce UI consistency across the application.

The frontend of MadChatter should have a coherent UI and make use of intuitive components. While
building such a UI from scratch is possible, it can drastically increase development times and the
number of bugs encountered. Thus, it seems logical to use a component library.

## Decided Approach

As a choice of component library, we have decided to use [shadcn](https://ui.shadcn.com/).
